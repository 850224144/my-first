# A股二买系统操作手册

> 版本：v1.0（方案 K / L / M 合并版）  
> 当前定位：自动扫描 + 自动计划 + 自动持仓跟踪 + 手动交易确认  
> 重要原则：本系统只做交易辅助，不做自动下单。

---

## 1. 系统定位

本系统用于扫描 A 股日线级别的趋势回踩二买候选，核心流程是：

```text
数据准备
→ 股票池构建
→ 盘中观察 observe
→ 尾盘确认 tail_confirm
→ 收盘复盘 after_close
→ 生成明日交易计划 trade_plan
→ 持仓跟踪 position_tracker
→ 交易结束归档 trade_journal
```

系统当前不负责自动买卖，只负责：

```text
1. 自动发现候选
2. 自动筛选风险
3. 自动生成交易计划
4. 自动跟踪持仓
5. 自动生成日报
6. 给出止损 / 止盈 / 时间止损提醒
```

交易执行仍由人工确认。

## 4. 核心数据文件说明

### 4.1 股票池

```text
data/universe.parquet
```

作用：保存当前可扫描股票池。

生成命令：

```bash
python run_scan.py --build-universe --workers 1
```

### 4.2 观察池

```text
data/watchlist.parquet
```

作用：保存盘中 observe 发现的候选，以及 tail_confirm / after_close 更新后的状态。

主要字段：

```text
code
name
first_seen_at
last_seen_at
mode
signal
total_score
risk_pct
action
entry_type
trigger_price
stop_loss
status
warnings
note
```

状态说明：

```text
active       继续观察
waiting      等待确认 / 等待突破
confirmed    尾盘确认
high_risk    风险过高，只观察
rejected     剔除
expired      过期
```

### 4.3 明日交易计划

```text
data/trade_plan.parquet
```

作用：保存 after_close 生成的明日关注计划。

只保存风险相对可接受、不是纯高风险观察的票。

### 4.4 持仓记录

```text
data/positions.parquet
```

作用：保存当前 open / closed 持仓。

### 4.5 交易归档

```text
data/trade_journal.parquet
```

作用：保存已经关闭的交易记录，用于后续复盘统计。

### 4.6 报告目录

```text
data/reports/
```

包括：

```text
daily_report_YYYY-MM-DD.md
position_report_YYYY-MM-DD.md
```

## 5. 日常推荐使用方式

### 5.1 正常自动模式

正式使用时，保持调度器运行即可。

启动后台调度器：

```bash
mkdir -p logs
nohup python run_scheduler.py >> logs/scheduler.out 2>&1 &
```

检查是否运行：

```bash
ps aux | grep run_scheduler.py | grep -v grep
```

查看调度器主日志：

```bash
tail -f logs/scheduler.out
```

查看具体任务日志：

```bash
ls logs/scheduler
tail -f logs/scheduler/$(date +%Y%m%d)_observe_morning.log
```

停止调度器：

```bash
ps aux | grep run_scheduler.py | grep -v grep
kill <PID>
```

## 6. 自动任务时间表

当前 APScheduler 自动任务安排：

```text
09:15  preflight / coverage 检查
09:35  observe_morning：盘中观察，扫描股票池，写入 watchlist
11:25  observe_midday：上午复扫，更新 watchlist
14:50  tail_confirm：尾盘确认，优先只扫 watchlist
17:30  after_close：收盘后更新缓存 + 重建股票池 + 收盘复盘 + 生成 trade_plan
20:30  daily_report：生成交易日报
22:30  night_cache_expand：夜间扩容历史K缓存
```

说明：

```text
15:40 不做正式日线更新。
17:30 才做正式收盘复盘和明日计划。
```

原因：15:40 某些数据源可能还没有完整更新，过早更新容易拿到不稳定数据。

## 7. 手动执行主流程

自动调度用于日常主线；下面命令用于调试、应急、复盘。

### 7.1 查看覆盖率

```bash
python run_scan.py --coverage
```

重点看：

```text
stock_daily 有效缓存股票
universe 股票池
realtime_quote 覆盖股票
```

### 7.2 扩容历史 K 缓存

小批量：

```bash
python run_scan.py --build-daily-cache --daily-limit 100 --daily-workers 1
```

正常扩容：

```bash
python run_scan.py --build-daily-cache --daily-limit 300 --daily-workers 1
```

建议：

```text
盘中不要大规模扩容历史K。
扩容历史K放在夜间或收盘后。
--daily-workers 保持 1，避免限流和数据库冲突。
```

### 7.3 构建股票池

```bash
rm -f data/universe.parquet
python run_scan.py --build-universe --workers 1
```

### 7.4 盘中观察

```bash
python run_scan.py --mode observe --workers 1
```

作用：

```text
1. 扫描 universe 股票池
2. 生成盘中观察候选
3. 写入 data/watchlist.parquet
```

注意：observe 不是买入信号，只是观察候选。

### 7.5 尾盘确认

```bash
python run_scan.py --mode tail_confirm --workers 1
```

作用：

```text
1. 优先读取 watchlist
2. 只扫描 active / waiting / high_risk 等候选
3. 更新尾盘状态
```

尾盘确认比 observe 更接近交易动作，但仍需结合风险、量能和触发价。

### 7.6 收盘复盘

```bash
python run_scan.py --mode after_close --workers 1
```

作用：

```text
1. 收盘后重新评分
2. 更新 watchlist
3. 生成 data/trade_plan.parquet
```

### 7.7 生成交易日报

```bash
python run_scan.py --daily-report
```

生成：

```text
data/reports/daily_report_YYYY-MM-DD.md
```

## 9. 持仓管理

持仓模块只负责记录和提醒，不自动买卖。

### 9.1 查看持仓

```bash
python run_positions.py --list
```

### 9.2 新增持仓

如果买入了比亚迪：

```bash
python run_positions.py --add-position 002594 --buy-price 103.5 --shares 100
```

如果 `data/trade_plan.parquet` 里已有该股票计划，系统会自动继承：

```text
stop_loss
take_profit_1
take_profit_2
```

手动指定止损和目标：

```bash
python run_positions.py --add-position 002594 \
  --buy-price 103.5 \
  --shares 100 \
  --stop-loss 96.73 \
  --target1 110.76 \
  --target2 116.38
```

如果同一股票已经有 open 持仓，需要覆盖：

```bash
python run_positions.py --add-position 002594 --buy-price 103.5 --shares 100 --force
```

### 9.3 跟踪持仓

```bash
python run_positions.py --track
```

生成：

```text
data/reports/position_report_YYYY-MM-DD.md
```

系统会判断：

```text
HOLD                继续持有
EXIT_STOP           跌破止损
REDUCE_TARGET1      达到目标1，建议减仓
TAKE_PROFIT_2       达到目标2，建议进一步止盈/清仓
TREND_BREAK_WARN    趋势破坏预警
TIME_WARN           持仓超过10天仍未脱离成本区
TIME_EXIT_WARN      持仓超过20天且收益不足
NO_DATA             无法获取当前价格
```

### 9.4 关闭持仓

止盈关闭：

```bash
python run_positions.py --close-position 002594 --exit-price 110 --exit-reason take_profit
```

止损关闭：

```bash
python run_positions.py --close-position 002594 --exit-price 96.5 --exit-reason stop_loss
```

手动关闭：

```bash
python run_positions.py --close-position 002594 --exit-price 105 --exit-reason manual_close
```

关闭后写入：

```text
data/trade_journal.parquet
```

## 11. 信号含义说明

### observe

```text
盘中观察候选。
只说明这只票有结构雏形，不代表可以买。
```

### tail_confirm

```text
尾盘确认。
要求更接近突破确认，适合小仓试错或加入次日计划。
```

### after_close

```text
收盘复盘。
生成明日计划，第二天不能高开追。
```

## 12. 操作建议含义

### 可执行

```text
评分高，风险可控，确认较充分。
但仍需人工确认盘口、量能和大盘环境。
```

### 可轻仓执行

```text
结构较好，但仍有瑕疵。
建议轻仓试错，不适合重仓。
```

### 轻仓观察 / 观察为主

```text
可以放入重点跟踪，但不应冲动买入。
```

### 等待突破

```text
尚未突破触发价。
只有突破触发价并且量能配合后，才考虑交易。
```

### 等待确认

```text
量能、突破或K线质量不足。
等待尾盘或次日确认。
```

### 只观察

```text
风险过高、涨幅过热或结构止损太远。
不建议开仓。
```

### 放弃

```text
触发强制否决条件。
```

## 13. 风险规则

当前系统核心风险逻辑：

```text
风险 <= 3%：低
3% < 风险 <= 5%：正常
5% < 风险 <= 8%：偏高，只能轻仓或等待确认
风险 > 8%：过高，不建议开仓
```

注意：

```text
分数高不代表可以买。
如果结构止损距离太远，系统会把它降级为“只观察”。
```

## 14. 交易生命周期示例

以一只观察候选为例：

```text
第 0 天 09:35 observe 发现候选，写入 watchlist
第 0 天 14:50 tail_confirm 检查是否突破、放量、不过热
第 0 天 17:30 after_close 生成明日 trade_plan
第 1 天 盘中触发价附近确认，人工决定是否买入
买入后 run_positions.py 录入持仓
持仓期间 position_tracker 跟踪止损、止盈、时间止损
卖出后 close-position 写入 trade_journal
```

标准持仓周期：

```text
快节奏：3-5 个交易日
正常节奏：5-15 个交易日
慢节奏：15-30 个交易日
```

如果 10 天还没有明显脱离成本区，要谨慎。

## 15. 常见问题排查

### 15.1 scheduler.out 不存在

只有用下面命令后台启动，才会生成：

```bash
nohup python run_scheduler.py >> logs/scheduler.out 2>&1 &
```

如果只是前台执行：

```bash
python run_scheduler.py
```

不会生成 `logs/scheduler.out`。

检查进程：

```bash
ps aux | grep run_scheduler.py | grep -v grep
```

### 15.2 watchlist 为空

先跑：

```bash
python run_scan.py --mode observe --workers 1
```

如果没有候选，说明当天没有符合 observe 条件的股票。

### 15.3 tail_confirm 扫描太多

正常情况下，`tail_confirm` 会优先读取 watchlist。若 watchlist 为空或和 universe 无交集，会回退扫描全 universe。

解决：

```bash
python run_scan.py --mode observe --workers 1
python run_scan.py --mode tail_confirm --workers 1
```

### 15.4 股票池太少

检查覆盖率：

```bash
python run_scan.py --coverage
```

如果有效缓存不足，扩容历史 K：

```bash
python run_scan.py --build-daily-cache --daily-limit 300 --daily-workers 1
```

然后重建股票池：

```bash
rm -f data/universe.parquet
python run_scan.py --build-universe --workers 1
```

### 15.5 候选很多但都是“只观察”

这通常不是 bug，而是系统认为：

```text
形态有雏形，但结构止损太远、风险太高、量能不足或当天过热。
```

这种情况不要追。

### 15.6 大盘 risk_off

表示大盘数据失败或风控关闭。系统原则是：

```text
大盘数据失败，默认不交易。
```

先检查 `core/market.py` 和新浪指数接口。

## 17. 当前系统边界

已经实现：

```text

自动交易计划
自动日报
手动持仓录入
```


```text
系统自动给候选和计划，人工做最后交易确认。
```

## 19. 最重要的执行原则

```text
observe：只观察，不买
waiting：等突破 / 等量能确认
tail_confirm：接近确认，但仍要看风险
after_close：生成明日计划，不追高
risk > 8%：不作为可执行买点
too_hot_today：不追
volume_not_confirm：等待
no_breakout：等待触发价
position_tracker：只提醒，不自动卖
```
