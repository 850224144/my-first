# 板块过滤 + 周线过滤补充说明

## 1. 过滤顺序

当前扫描顺序：

```text
大盘过滤
→ 股票池过滤
→ 板块强度过滤
→ 周线趋势过滤
→ 日线二买评分
→ 交易计划
→ 推送
```

本补丁不修改二买评分模型，只增加过滤层。

## 2. 板块过滤

板块强度评分使用：

```text
5日涨幅：25分
10日涨幅：25分
上涨占比：20分
站上20日线占比：20分
成交额放大：10分
```

强势市场默认取前 20% 板块；震荡市场取 market.py 中返回的 `sector_top_pct`。

## 3. 周线过滤

周线过滤只读本地 `stock_daily`，不会远程拉历史 K。

周线 V1 条件：

```text
至少30根周线
周线 MA5 > MA10 > MA20
最新周收盘价 > MA10
MA10 不向下
最近3周不明显创新低（允许2%噪音）
```

周线过滤用于避免“日线像二买，但周线仍是下跌趋势”的股票。

## 4. 建议命令

```bash
python run_scan.py --coverage
python run_scan.py --build-universe --workers 1
python run_scan.py --mode observe --workers 1
```

如果周线过滤后无股票，先不要急着放宽条件。先确认：

```bash
python run_scan.py --coverage
```

如果日线覆盖率很低，说明需要继续缓存更多股票：

```bash
python run_scan.py --build-daily-cache --daily-workers 1 --daily-limit 500
python run_scan.py --build-universe --workers 1
```

## 5. 注意

- 板块过滤依赖 `data/sector_members.parquet`；第一次刷新会较慢。
- 周线过滤依赖 `stock_daily`，缓存不足会导致过滤后股票偏少。
- 本阶段目标是先提升信号质量，不追求信号数量。
