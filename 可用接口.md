# 总原则：两个接口体系不要混在一起

## 1. 选股宝接口定位

选股宝接口更适合做：

- 市场情绪
- 涨停股池
- 跌停股池
- 炸板股池
- 连板股池
- 强势股池
- 昨日涨停表现
- 题材、概念、涨停原因
- 快讯、政策、行业消息
- 龙头识别辅助

一句话：

> 选股宝更适合判断“今天市场在炒什么、谁是前排、谁是龙头”。

## 2. AkShare 接口定位

AkShare 更适合做：

- 全市场基础股票池
- A 股历史 K 线
- A 股实时行情
- 指数行情
- 行业/概念板块行情
- 板块成份股
- 资金流
- 龙虎榜
- 热度榜
- 基础排雷数据

一句话：

> AkShare 更适合补充基础行情、历史行情、板块、资金流和人气数据。

## 3. 项目里不要这样做

不要把选股宝和 AkShare 混成一个数据源：

```text
错误做法：
get_market_data() 里面一会儿请求选股宝，一会儿请求 AkShare，一会儿又请求腾讯/新浪。
```

这样后面会出现：

- 字段命名混乱
- 股票代码格式混乱
- 时间戳格式混乱
- 缓存难做
- 报错难定位
- 同一只股票重复提醒
- 策略不知道信号来源

## 4. 推荐的数据源分层

```text
选股宝：情绪层 / 题材层 / 龙头层
AkShare：辅助数据层 / 板块层 / 资金流层 / 人气层
腾讯/新浪：基础行情层 / K线层 / 实时价格层
```

# 第一部分：选股宝接口

## 1. 统一说明

### 1.1 基础域名

```text
https://flash-api.xuangubao.cn
```

### 1.2 返回格式

```json
{
  "code": 20000,
  "msg": "success",
  "data": []
}
```

### 1.3 股票代码格式

选股宝要求股票代码带交易所后缀：

| 市场 | 格式 | 示例 |
|---|---|---|
| 上交所 | `.SH` | `600519.SH` |
| 深交所 | `.SZ` | `000001.SZ` |
| 北交所 | `.BJ` | `834765.BJ` |

### 1.4 请求频率建议

```text
建议单 IP 每秒不超过 10 次请求。
项目中建议更保守：每秒 2~5 次，失败后指数退避。
```

### 1.5 历史数据范围

```text
支持查询近 1 年历史数据。
```

## 2. 市场概览接口

### 2.1 当日市场情绪数据

```http
GET https://flash-api.xuangubao.cn/api/market_indicator/line?fields=rise_count,fall_count,flat_count,limit_up_count,limit_down_count,limit_up_broken_count,limit_up_broken_ratio
```

### 2.2 功能

获取当日全市场涨跌、涨停、跌停、炸板数据。

### 2.3 可选字段

| 字段 | 含义 | 项目用途 |
|---|---|---|
| `rise_count` | 上涨家数 | 判断市场是否普涨 |
| `fall_count` | 下跌家数 | 判断亏钱效应 |
| `flat_count` | 平盘家数 | 辅助统计 |
| `limit_up_count` | 涨停家数 | 判断短线情绪强弱 |
| `limit_down_count` | 跌停家数 | 判断极端亏钱效应 |
| `limit_up_broken_count` | 炸板家数 | 判断封板失败数量 |
| `limit_up_broken_ratio` | 炸板率，小数 | 判断短线接力风险 |

### 2.4 历史市场情绪数据

```http
GET https://flash-api.xuangubao.cn/api/market_indicator/line?fields=rise_count,fall_count,limit_up_count,limit_down_count,limit_up_broken_count,limit_up_broken_ratio&date=2026-04-28
```

### 2.5 项目建议

用于 `market_emotion_score`：

```text
rise_count / fall_count
limit_up_count
limit_down_count
limit_up_broken_ratio
```

初始过滤规则可以先写成：

```text
limit_down_count <= 20
limit_up_broken_ratio <= 0.35
rise_count / max(fall_count, 1) >= 0.8
```

如果不满足，龙头援军模式只输出观察，不输出买入计划。

## 4. 涨停股池：`limit_up`

### 4.1 接口

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=limit_up
```

### 4.2 功能

获取当日所有涨停股票详细信息。

### 4.3 核心返回字段

| 字段 | 含义 | 项目用途 |
|---|---|---|
| `symbol` | 股票代码，如 `600519.SH` | 统一代码 |
| `stock_chi_name` | 股票名称 | 提醒展示 |
| `price` | 最新价 | 实时判断 |
| `change_percent` | 涨跌幅 | 强度判断 |
| `limit_up_days` | 连板天数 | 龙头评分核心字段 |
| `first_limit_up_time` | 首次涨停时间戳 | 越早越强 |
| `last_limit_up_time` | 最后涨停时间戳 | 回封判断 |
| `surge_reason` | 涨停原因，含板块/概念 | 题材解析核心字段 |
| `turnover_ratio` | 换手率 | 流动性和分歧判断 |
| `circulating_market_cap` | 流通市值，亿元 | 排除过小/过大异常票 |

### 4.4 项目用途

主要用于：

```text
1. 识别当天主线题材
2. 识别题材前排
3. 识别连板高度
4. 识别首次涨停时间
5. 解析 surge_reason，生成 theme 标签
6. 构建 leader_score
```

## 5. 其他股票池

| 股票池名称 | `pool_name` 参数 | 功能 | 项目用途 |
|---|---|---|---|
| 跌停股池 | `limit_down` | 获取当日跌停股票 | 判断亏钱效应、排雷 |
| 炸板股池 | `limit_up_broken` | 获取当日炸板股票 | 判断接力失败、炸板风险 |
| 昨日涨停股池 | `yesterday_limit_up` | 获取昨日涨停股票今日表现 | 判断昨日涨停溢价 |

| 强势股池 | `strong_stock` | 近期强势股票 | 龙头候选辅助 |
| 连板股池 | `continuous_limit_up` | 连板股票，通常 ≥2 板 | 龙头识别核心 |

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=limit_down
```

用途：


### 5.2 炸板股池


用途：

```text
识别高风险题材。
```

### 5.3 昨日涨停股池

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=yesterday_limit_up
```

用途：

```text
判断昨日涨停今日是否有溢价。
用于判断短线情绪是否继续修复。
```

### 5.4 强势股池

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=strong_stock
```

用途：

```text
辅助确认龙头、核心前排、趋势强势票。
```

### 5.5 连板股池

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=continuous_limit_up
```

用途：

```text
龙头评分核心。
同题材连板高度最高者优先进入 leader_candidates。
```

### 5.6 次新股池

```http
GET https://flash-api.xuangubao.cn/api/pool/detail?pool_name=new_stock
```

用途：


## 9. 选股宝 Python 调用示例

```python
import requests
from datetime import datetime

BASE_URL = "https://flash-api.xuangubao.cn"


def request_xgb(path: str, params: dict | None = None, timeout: int = 10):
    url = BASE_URL + path
    try:
        resp = requests.get(url, params=params, timeout=timeout)
        resp.raise_for_status()
        data = resp.json()
        if data.get("code") != 20000:
            raise RuntimeError(f"XGB接口失败: {data.get('msg')}")
        return data.get("data", [])
    except Exception as exc:
        print(f"请求选股宝异常: {url}, err={exc}")
        return []


def get_market_indicator(date: str | None = None):
    fields = ",".join([
        "rise_count",
        "fall_count",
        "flat_count",
        "limit_up_count",
        "limit_down_count",
        "limit_up_broken_count",
        "limit_up_broken_ratio",
    ])
    params = {"fields": fields}
    if date:
        params["date"] = date
    return request_xgb("/api/market_indicator/line", params=params)


def get_pool(pool_name: str, date: str | None = None):
    params = {"pool_name": pool_name}
    if date:
        params["date"] = date
    return request_xgb("/api/pool/detail", params=params)


def get_limit_up_stocks(date: str | None = None):
    return get_pool("limit_up", date=date)


def get_stock_data(symbols: list[str]):
    fields = ",".join([
        "symbol",
        "stock_chi_name",
        "price",
        "change_percent",
        "open",
        "high",
        "low",
        "volume",
        "amount",
        "turnover_ratio",
    ])
    params = {
        "symbols": ",".join(symbols[:20]),
        "fields": fields,
    }
    return request_xgb("/api/stock/data", params=params)


if __name__ == "__main__":
    market = get_market_indicator()
    print("市场概览:", market)

    limit_up = get_limit_up_stocks()
    print(f"今日涨停家数: {len(limit_up)}")
    for item in limit_up[:5]:
        print(item.get("stock_chi_name"), item.get("symbol"), item.get("limit_up_days"))
```

# 第二部分：AkShare 接口

## 1. 统一说明

AkShare 是 Python 金融数据接口库，调用方式一般为：

```python
import akshare as ak

df = ak.stock_zh_a_hist(symbol="600519", period="daily", start_date="20240101", end_date="20260429", adjust="qfq")
print(df)
```

## 2. AkShare 使用注意事项

```text
1. AkShare 的底层数据源很多来自东方财富、腾讯、新浪、同花顺、雪球等。
2. 不同函数返回字段不统一，必须做字段标准化。
3. 不同函数股票代码格式不统一，有的要 600519，有的要 sh600519，有的返回 600519.SH。
4. 不能高频请求，尤其是东方财富相关接口，容易出现远端断开、连接失败、限流等问题。
5. 项目里必须做 timeout、重试、失败降级、本地缓存。
6. AkShare 不建议作为唯一实时数据源，可作为补充数据源。
```

| `stock_sz_a_spot_em` | 东方财富 | 深 A 实时行情 | 可用于分市场扫描 |
| `stock_bj_a_spot_em` | 东方财富 | 北交所实时行情 | 可用于北交所扩展 |
| `stock_new_a_spot_em` | 东方财富 | 新股实时行情 | 默认不参与援军模式 |
| `stock_kc_a_spot_em` | 东方财富 | 科创板实时行情 | 可选 |
| `stock_zh_b_spot_em` | 东方财富 | B 股实时行情 | 不建议纳入本项目 |
| `stock_zh_a_daily` | 多源 | A 股日频历史行情 | 可用，但优先级低于 `stock_zh_a_hist` |
| `stock_zh_a_minute` | 多源 | A 股分钟历史行情 | 盘中结构可选 |
| `stock_zh_a_cdr_daily` | 多源 | CDR 日频历史行情 | 暂不纳入 |

| `stock_zh_a_hist_tx` | 腾讯 | A 股日频历史行情 | 与项目腾讯源一致，可做备用 |

| `stock_zh_a_hist_pre_min_em` | 东方财富 | A 股盘前分时 | 暂不作为核心 |

### 示例：获取 A 股日 K

```python
import akshare as ak

hist_df = ak.stock_zh_a_hist(
    symbol="600519",
    period="daily",
    start_date="20240101",
    end_date="20260429",
    adjust="qfq",
)
print(hist_df.tail())
```

# 最后建议

对当前项目来说，不要把接口堆满。第一版只做这个闭环：

```text
选股宝判断：市场情绪 + 主线题材 + 龙头地位
腾讯/新浪判断：K线结构 + 买入区 + 撤军线 + 目标位
AkShare补充：交易日历 + 股票池 + 板块成份 + 人气/龙虎榜辅助
```

这样做最稳。  
接口越多，不代表系统越强。真正重要的是：

```text
同一只股票为什么入选？
它是不是板块龙头？
它有没有援军位置？
错了在哪里撤？
对了看到哪里？
盈亏比是否合格？
```
